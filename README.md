# Research_Prompt

# 
# Role
你是一位世界顶尖的学术插画专家，专注于为计算机视觉与人工智能领域的顶级会议（如 CVPR, NeurIPS, ICLR）绘制高质量、直观且美观的论文特色与创新图， 现在为@科研计划书.docx的{四、特色与创新之处（建议 500字以内）}绘制图，要结合里面的内容来，不要脱离了里面的内容。 

# Task
请模仿我提供的【参考图】，首先深刻理解其配色风格、画图轮廓线的粗细不一、字体风格、曲线的风格。然后，基于你的理解，设计并绘制一张专业的特色与创新图。  
特色与创新之处:
{
本研究的特色与创新主要体现在四个方面：一是研究视角从“事后检测”前移到“事前推演”，不只在风险发生后识别问题，而是尝试面向长时程、多分支决策过程提前预测多智能体行为轨迹与风险演化路径，从而提升系统的前瞻性治理能力。二是将AI安全与未来预测统一为同一研究问题，把风险识别、演化建模与干预决策放在闭环框架中考虑，使模型既能判断“是否存在风险”，也能回答“风险如何形成、如何传播、如何被抑制”。三是以代码安全和可信智能体场景作为可验证起点，在数字环境中构建可观测、可复现、可量化评测的实验体系，并将工具调用链、环境反馈和群体协同行为纳入统一建模，再进一步探索向更一般多智能体复杂系统迁移的可能。四是强调原创模型、可解释风险演化和可执行干预三者统一，结合世界模型、因果推理与安全约束学习，力求让模型既能“预测未来”，又能“解释未来”“改变未来”，形成兼具方法创新、评测价值与应用潜力的研究方案。}
当然要结合上述内容还有你自己已知外部的研究，不仅仅局限于这里的内容，说明清楚就行。

# Visual Constraints
1. 风格基调：
   - 必须具备顶会论文风格：专业、干净、现代、极简主义。
   - 核心美学：采用扁平化矢量插画风格，线条简洁，参考 DeepMind 或 OpenAI 论文中的图表美学。
   - 拒绝卡通感、油画感或过度艺术化，保持严谨的学术图表美学。
   - 背景必须是纯白色，无任何纹理或阴影。
   - 假如图中涉及圆或者椭圆，每个圆/椭圆轮廓线的粗细要不一样，模拟人手画的过程，也就是画到一半，突然觉得不圆，又重新描画了一遍，导致粗细不一。

2. 色彩体系：
   - 严格使用淡色系或柔和色调。
   - 严禁使用过于鲜艳饱和的颜色（如大红大绿）或过于暗淡沉重的颜色。利用颜色的深浅变化来区分不同的模块类型。

3. 内容与布局：
   - 将理解到的方法论转化为清晰的模块和数据流箭头。
   - 适当使用现代、简洁的矢量图标嵌入到模块中，以增强直观性。

4. 文字规范：
   - 图中所有文字必须使用中文(专有名词，专业论文标题简称等可以保留英文)。 
   - 你必须为方法论中提到的关键模块或方程式添加清晰易读的文本标签。
   - 严禁在图中出现长句子、描述性段落或复杂的公式。文字是用来说明模块身份的，不是用来解释原理的。

5. 禁止事项：
   - 不允许使用逼真照片感。

   - 不允许难以辨认的文本。
   - 不允许廉价的 3D 阴影瑕疵。
